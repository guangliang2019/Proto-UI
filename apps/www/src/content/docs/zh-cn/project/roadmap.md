# 路线图

Proto UI 当前处于 v0 阶段。CLI 上线与首次公开发布目前都暂定为 **2026 年 5 月 14 日**。

这一阶段的重点不是尽快扩张功能表面，而是先完成协议层的基础闭环：让核心信息通路、执行语义与跨宿主翻译关系具备稳定、可验证的基础。

## 当前阶段：v0

v0 是 Proto UI 从概念走向可验证实现的阶段。

在这个阶段，Proto UI 主要围绕以下目标推进：

- 收敛组件原型的基础表达方式
- 稳定原型、适配器与运行时之间的核心语义
- 通过契约测试验证不同实现之间的一致性
- 在多个宿主环境中验证 Prototype 的落地能力
- 为基础原型库与适配器生态建立可依赖的参考基线

v0 的意义，不在于提供一个已经完备的 UI 框架，而在于确认 Proto UI 这套协议模型能够成立、能够落地，并具备继续扩展的基础。

## 下一阶段：v1

在 v1 阶段，Proto UI 将从“协议可验证”进一步走向“稳定可依赖”。

相比 v0，v1 的重点将包括：

- 为原型语法提供明确的稳定性与向后兼容承诺
- 将 Compiler 发展为更主要的执行路径
- 将 Proto UI 的落地目标从 Web 扩展到更广泛的平台
- 继续沉淀官方原型库、适配器与相关工具链
- 让协议语义、生成能力与生态协作方式形成更完整的闭环

如果说 v0 主要回答的是“Proto UI 是否成立”，那么 v1 更关注“Proto UI 如何稳定地走向更广泛的实践”。

## 当前阶段不承诺的内容

在 v0 阶段，Proto UI 还不承诺以下内容：

- 工业级别的向后兼容保证
- 完整的性能优化
- 成熟完备的适配器生态
- 已覆盖所有目标平台的稳定实现

这些能力并不是不重要，而是需要建立在协议语义已经收敛、基础约束已经明确的前提之上。

## 总结

Proto UI 的演进路径，不是先追求功能堆叠，再回头整理基础。

相反，它会先把协议层语义、适配关系与实现边界收敛清楚，再逐步扩展稳定性、生成能力与生态规模。

v0 关注“这套模型能否成立并落地”。  
v1 关注“这套模型如何稳定地扩展并长期存在”。
