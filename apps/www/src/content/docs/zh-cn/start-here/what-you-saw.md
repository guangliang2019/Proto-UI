## 你刚刚看到的是什么？

在 [Proto UI 官网首页](https://www.proto-ui.com) 的 Demo 中，你可以在 React、Vue、Web Component 等不同框架之间切换，而组件的交互行为保持完全一致。

这不是视觉模拟，也不是简单的样式切换。  
每一次切换，都会重新加载对应框架的运行时环境。

同一份交互定义，被不同宿主分别执行。

> 如果你还没有体验首页 Demo，建议先回到首页看一眼。  
> 它不是阅读本文的前置条件，但会帮助你更快理解 Proto UI 到底在展示什么。

## 这一现象为什么值得在意？

- 你看到的不是“一套组件适配了多个框架”这么简单。
- 你看到的是：组件的交互语义可以先被独立表达，再由不同宿主分别落地。
- Proto UI 想证明的，不是跨框架兼容技巧，而是一层独立交互协议的可能性。

## 在 Proto UI 中，最小的理解单位是什么？

- **Prototype（原型）**：描述组件做什么。
- **Adapter（适配器）**：决定这份原型如何在某个宿主中被解释和运行。

在这篇里，先记住这两个词就够了。
