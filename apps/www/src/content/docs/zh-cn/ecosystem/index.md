## 这篇文章要回答什么？

- Proto UI 当前的生态由哪些层组成。
- 哪些内容已经存在，哪些内容仍在建设中。
- 如果有人想参与生态，应该先理解哪一层。

## 这篇适合什么读者？

写作提示：

- 面向想理解 Proto UI 生态全貌的人。
- 面向可能参与原型、适配器或测试建设的人。
- 不面向只想把 Proto UI 装进项目的普通使用者。

## 生态这一章负责什么？

写作提示：

- 介绍原型库、适配器库、生态建设方向与参与入口。
- 解释“生态建设”与“工程实现”的区别。

## 生态这一章不负责什么？

写作提示：

- 不替代安装与使用文档。
- 不替代工程实现章节。
- 不替代协作流程细则。

## 建议如何阅读这一章？

写作提示：

- 给出阅读顺序，例如先看原型库概览与适配器库概览，再决定是否阅读“编写原型 / 编写适配器”。

## 下一步怎么读？

- 如果想看当前已有能力：前往 [原型库（概览）](/zh-cn/ecosystem/prototypes/) 与 [适配器库（概览）](/zh-cn/ecosystem/adapters/)
