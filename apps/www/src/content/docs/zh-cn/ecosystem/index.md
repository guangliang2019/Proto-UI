## 这篇文章要回答什么？

- Proto UI 生态里现在有哪些角色、产物和参与方式。
- 官方主要维护什么，社区又可以从哪些位置进入。
- 如果有人想参与 Proto UI，最适合先从哪里开始。

## 这篇适合什么读者？

- 面向想理解 Proto UI 生态全貌的人。
- 面向读完白皮书后开始考虑参与项目的人。
- 面向还不确定自己更适合做原型、适配器、文档还是测试的人。
- 不面向只想把 Proto UI 装进项目的普通使用者。

## Proto UI 生态里有哪些角色？

写作提示：

- 介绍官方原型、官方适配器、社区贡献者、文档与测试维护者这些角色。
- 说明不同角色面对的是不同层面的生态工作，而不只是“写代码”。

## 生态里有哪些主要产物？

写作提示：

- 介绍原型、适配器、契约与测试、文档、Demo、工具这些产物分别承担什么作用。
- 说明它们之间不是并列模块，而是一条相互依赖的生态链。

## 官方维护什么，社区又可以做什么？

写作提示：

- 说明官方更关注哪些基础部分。
- 说明社区可以从原型、适配器、文档、Demo、测试、讨论等不同方向参与。

## 这一章与其他章节如何分工？

- 不替代 `Start Here`
- 不替代 `Whitepaper`
- 不替代 `Specifications` 与 `Engineering`
- 它的重点是把“理解生态”和“进入贡献”这件事打通

## 建议如何阅读这一章？

- 如果你还不知道怎么参与：先看 `如何参与贡献`
- 如果你已经确定想做什么：直接进入 `编写原型` 或 `编写适配器`
- 如果你只是想知道目前生态长什么样：再去看 `原型库（概览）` 与 `适配器库（概览）`
