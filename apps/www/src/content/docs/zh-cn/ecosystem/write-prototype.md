## 这篇文章要回答什么？

写作提示：

- 说明什么样的前提下才适合开始编写新的原型。
- 说明编写原型在 Proto UI 中意味着什么责任，而不是仅仅新增一个组件实现。

## 这篇适合什么读者？

写作提示：

- 面向已经理解 Proto UI 哲学、规范与现有原型库的人。
- 不面向首次使用 Proto UI 的读者。

## 写原型之前应先确认什么？

写作提示：

- 是否真的需要新增原型，而不是组合现有原型。
- 是否理解信息通路模型与原型拆分原则。
- 是否理解编写原型会对生态语义产生影响。

## 这里应当包含哪些信息？

写作提示：

- 原型设计的判断标准
- 与白皮书、规范、工程 API 的前置关系
- 何时应该进入契约与测试
