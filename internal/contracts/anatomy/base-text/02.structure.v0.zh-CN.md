# Anatomy 结构声明底本（v0）

本文定义 Anatomy v0 的最小结构声明模型，包括：

- `family`
- `profile`
- `claim`
- `role`
- `relation`
- `requirement`

本文只讨论**结构声明语义**，不展开 runtime 查询与诊断分级细节。

---

## 1. 设计目标

Anatomy 的结构声明层需要同时满足以下目标：

- 提供稳定的 family 级语义坐标
- 允许具体 prototype 通过 profile 收紧规则
- 允许 prototype instance 在 runtime 中认领某个 role
- 保持可拓展，但不滑向任意 override / merge 系统
- 明确排除自动装配、自动补全、自动重写结构

---

## 2. Family Token

Anatomy Family 应通过一个稳定的、可共享的 token 标识。

建议形式：

```ts
const SELECT_FAMILY = createAnatomyFamily('select');
```

其语义地位应接近 `ContextKey`：

- 用于 identity
- 可跨文件共享
- 可作为 claim / 查询 / 诊断的统一引用对象

`debugName`：

- 仅用于调试与诊断
- 不影响 identity

Family token 本身不承载完整规则定义；规则由 `def.anatomy.family(...)` 注册。

---

## 3. Family 声明

Family 是 Anatomy 的基线声明。

它必须定义：

- `roles`
- `relations`
- 可选的 `profiles`

建议形态：

```ts
def.anatomy.family(SELECT_FAMILY, {
  roles: {
    root: {
      cardinality: { min: 1, max: 1 },
      requires: [{ kind: 'hook', name: 'asSelect' }],
    },
    trigger: {
      cardinality: { min: 0, max: Infinity },
      requires: [{ kind: 'hook', name: 'asTrigger' }],
    },
    content: {
      cardinality: { min: 0, max: Infinity },
      requires: [{ kind: 'hook', name: 'asFloating' }],
    },
    item: {
      cardinality: { min: 0, max: Infinity },
      requires: [{ kind: 'hook', name: 'asSelectItem' }],
    },
  },
  relations: [
    { kind: 'contains', parent: 'root', child: 'trigger' },
    { kind: 'contains', parent: 'root', child: 'content' },
    { kind: 'contains', parent: 'content', child: 'item' },
  ],
});
```

### 3.1 roles

`roles` 是 family 内全部可识别结构角色的声明表。

每个 role 的声明至少包含：

- `cardinality`
- 可选的 `requires`

v0 中，role 名称应为字符串。

### 3.2 cardinality

`cardinality` 用于描述一个 role 在某个 anatomy domain 内允许出现的数量范围。

语法：

```ts
{ min: number, max: number | Infinity }
```

规则：

- `min` 和 `max` 必须存在
- `min` 必须大于等于 `0`
- `max` 必须大于等于 `min`
- `max` 允许为 `Infinity`

`cardinality` 使用对象形式而不是 tuple，原因是：

- 可读性更强
- 更便于后续扩展
- 更适合 profile 的 partial refinement

### 3.3 requires

`requires` 表示某个 role 必须满足的额外结构要求。

v0 中，`requires` 先只支持对 `as-hook` 的检查。

建议形态：

```ts
requires: [{ kind: 'hook', name: 'asTrigger' }];
```

语义：

- role 的认领者若缺失这些 hook，应被视为结构不满足该 role 的要求
- `requires` 属于结构检查，不属于行为注入

### 3.4 relations

`relations` 用数组描述 role 与 role 之间的结构关系。

v0 建议先支持：

- `kind: 'contains'`

示例：

```ts
relations: [
  { kind: 'contains', parent: 'root', child: 'content' },
  { kind: 'contains', parent: 'content', child: 'item' },
];
```

`relations` 使用数组而不是对象嵌套，原因是：

- 不把语法绑定到单一树状结构
- 更利于后续扩展非嵌套关系
- 更利于 profile 叠加与诊断定位

v0 不要求一次性支持所有 relation 类型，但应保留未来加入其他关系的空间。

## 4. Profile 声明

Profile 是建立在 family 之上的**具名收紧规则集**。

它用于表达：

- 某个具体 prototype 默认期望的 anatomy
- 某种常见交互变体的结构偏好

Profile 可以是 family 声明中的一部分。

建议形态：

```ts
def.anatomy.family(SELECT_FAMILY, {
  roles: {
    root: { cardinality: { min: 1, max: 1 } },
    trigger: { cardinality: { min: 0, max: Infinity } },
    content: { cardinality: { min: 0, max: Infinity } },
    item: { cardinality: { min: 0, max: Infinity } },
  },
  relations: [
    { kind: 'contains', parent: 'root', child: 'trigger' },
    { kind: 'contains', parent: 'root', child: 'content' },
    { kind: 'contains', parent: 'content', child: 'item' },
  ],
  profiles: {
    default: {
      roles: {
        trigger: { cardinality: { max: 1 } },
        content: { cardinality: { max: 1 } },
      },
    },
    range: {
      roles: {
        trigger: { cardinality: { max: 2 } },
        content: { cardinality: { max: 2 } },
      },
    },
  },
});
```

### 4.1 profile 的本质

Profile 是 refinement，而不是 override。

它的职责是：

- 收紧 family 已有规则
- 补充 family 未指定的更严格期望

它不应：

- 放宽 family 基线
- 删除 family 的结构语义
- 推翻 family 的 invariant

### 4.2 profile 的 partial 特性

Profile 应允许只声明需要收紧的那部分规则。

未声明的内容应继承 family 基线。

例如：

```ts
profiles: {
  default: {
    roles: {
      trigger: { cardinality: { max: 1 } },
    },
  },
}
```

表示：

- `trigger.max` 被收紧到 `1`
- 其他 role 与 relation 保持 family 语义

### 4.3 profile 可做的收紧

Profile 在 v0 中建议只允许以下类型的收紧：

- 收紧 `cardinality`
- 新增 `requires`
- 补充更严格的 `relations`

具体要求：

- `cardinality.min` 只能升高或保持
- `cardinality.max` 只能降低或保持
- `requires` 只能新增，不能删除
- `relations` 只能补充更严格关系，不能取消 family 已有关系

## 5. Claim

Claim 用于声明“当前 prototype instance 在某个 family 中承接什么 role”。

建议形态：

```ts
def.anatomy.claim(SELECT_FAMILY, {
  role: 'item',
});
```

根 part 可附带 profile：

```ts
def.anatomy.claim(SELECT_FAMILY, {
  role: 'root',
  profile: 'default',
});
```

### 5.1 claim 的职责

Claim 只负责：

- 把当前 instance 归入某个 family
- 声明当前 instance 承接哪个 role
- 在 root 的情况下，声明当前 domain 采用哪个 profile

Claim 不负责：

- 创建别的 part
- 自动连接缺失 part
- 修改现有树结构

### 5.2 claim 的最小规则

v0 中建议至少满足以下规则：

- 同一 instance 在同一 family 中只能 claim 一个 role
- 同一 domain 中只能有一个 `root` role part
- `profile` 只允许在 `root` claim 时指定
- 非 `root` part 若指定 `profile`，应视为非法声明

### 5.3 domain 的 profile 来源

一个 anatomy domain 的 profile 应由其 `root` part 决定。

若 `root` 未指定 profile，则该 domain 只按 family 规则解释。

其他 part 只加入当前 domain，不单独声明 profile。

---

## 6. 合并与扩展边界

Anatomy v0 需要支持“family 宽松、profile 收紧”的模式，但不应开放任意 override 系统。

因此建议明确以下边界：

- 允许：family 基线 + profile refinement
- 允许：profile partial 声明
- 不允许：profile 放宽 family
- 不允许：通用 override / replace 整个 family
- 不允许：运行时动态改写 family / profile 定义

这意味着 Anatomy v0 的结构声明模型是：

- 可分层
- 可收紧
- 可局部补充
- 但不可任意重写

## 7. 最小示例

```ts
const SELECT_FAMILY = createAnatomyFamily('select');

def.anatomy.family(SELECT_FAMILY, {
  roles: {
    root: {
      cardinality: { min: 1, max: 1 },
      requires: [{ kind: 'hook', name: 'asSelect' }],
    },
    trigger: {
      cardinality: { min: 0, max: Infinity },
      requires: [{ kind: 'hook', name: 'asTrigger' }],
    },
    content: {
      cardinality: { min: 0, max: Infinity },
      requires: [{ kind: 'hook', name: 'asFloating' }],
    },
    item: {
      cardinality: { min: 0, max: Infinity },
      requires: [{ kind: 'hook', name: 'asSelectItem' }],
    },
  },
  relations: [
    { kind: 'contains', parent: 'root', child: 'trigger' },
    { kind: 'contains', parent: 'root', child: 'content' },
    { kind: 'contains', parent: 'content', child: 'item' },
  ],
  profiles: {
    default: {
      roles: {
        trigger: { cardinality: { max: 1 } },
        content: { cardinality: { max: 1 } },
      },
    },
  },
});

def.anatomy.claim(SELECT_FAMILY, {
  role: 'root',
  profile: 'default',
});
```

---

## 8. 非目标

本文不定义：

- runtime 查询 API 的最终形态
- 诊断等级的完整规则
- 自动补全默认结构
- 自动装配或默认 part materialization
