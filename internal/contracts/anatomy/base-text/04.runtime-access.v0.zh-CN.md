# Anatomy Runtime Access 底本（v0）

本文定义 Anatomy v0 的 runtime 受限访问语义。

Anatomy 的 runtime access 仅用于：

- 查询当前 anatomy domain 中哪些 role / part 实际存在
- 读取这些 part 已显式 expose 的能力或状态

它不是通用通信系统，也不是实例内部能力的直接泄漏通道。

---

## 1. 设计目标

Anatomy runtime access 应满足：

- 允许同一 anatomy domain 内的 part 发现彼此
- 保持访问能力受限
- 避免演变为新的 instance registry
- 避免替代 `context`、`event`、`props`

---

## 2. 基本原则

### 2.1 仅限同一 Family / Domain

`run.anatomy` 的查询范围仅限：

- 当前 instance 所属的 anatomy family
- 当前 instance 所在的 anatomy domain

不应允许任意跨 domain、跨 family 搜索。

### 2.2 仅观察实际存在的 part

`run.anatomy` 只能返回当前 runtime 中**实际存在**的 parts。

Anatomy 不负责：

- 虚拟补全缺失 role
- 伪造默认 part
- 隐式 materialize 一个“应当存在但实际上不存在”的 part

### 2.3 访问面受限

`run.anatomy` 返回的不是实例引用，而是受限的观察句柄。

该句柄只应用于：

- 查询 role
- 查询 expose 是否存在
- 读取 expose 值

不应用于：

- 访问内部状态内核
- 访问 host 节点
- 调用未 expose 的内部能力

---

## 3. 建议的最小查询面

建议至少提供以下能力：

```ts
run.anatomy.has(FAMILY, 'content');
run.anatomy.parts(FAMILY);
run.anatomy.partsOf(FAMILY, 'item');
```

语义：

- `has`：某 role 是否至少存在一个实际 part
- `parts`：返回当前 domain 内该 family 的全部可观察 parts
- `partsOf`：返回当前 domain 内某 role 的全部可观察 parts

如果当前 instance 不属于该 family / domain，相关调用应失败或返回明确不可用结果。

---

## 4. Part View

`run.anatomy` 返回的 part 应是 **Part View**，而不是实例句柄。

建议最小形态：

```ts
type AnatomyPartView = {
  role: string;
  hasExpose(key: string): boolean;
  getExpose(key: string): unknown | null;
};
```

可选扩展：

```ts
type AnatomyPartView = {
  role: string;
  hasExpose(key: string): boolean;
  getExpose(key: string): unknown | null;
  hasHook(name: string): boolean;
};
```

其中：

- `role` 用于说明该 part 的 anatomy 角色
- `hasExpose` / `getExpose` 只面向显式 `def.expose(...)` 的结果
- `hasHook` 如提供，也应仅作为受限观察能力

---

## 5. Expose 访问语义

Anatomy runtime access 不引入新的输出系统。

它主要复用 expose 作为可观察面，因此应遵循以下原则：

- 只能访问显式 expose 的内容
- 不得透传实例对象本身
- 不得绕过 expose 访问未公开能力

这意味着：

- 某个 `content` role 若 expose 了 `close`，其他 part 可以读取到它
- 某个 part 若未 expose 某项能力，Anatomy 不应替其推断或补造

---

## 6. 生命周期与失效

`run.anatomy` 返回的 Part View 只对当前实际存在的 part 有意义。

若对应 part 已 dispose 或离开当前 domain，则：

- 该 view 应被视为失效，或
- 相关访问应返回明确的不可用结果，或
- 相关调用应抛出可区分错误

v0 不强制规定最终实现形态，但必须保证：

- 不会把已失效 part 伪装成仍可正常访问

---

## 7. 推荐的边界约束

为避免 `run.anatomy` 演变为隐式实例注册表，建议至少遵循：

- 只允许查询已声明 family 的 role
- 只允许读取已 expose 的能力
- 不返回原始实例引用
- 不返回 host 引用
- 不提供通用 update / emit / context 透传

因此，不推荐提供以下能力：

- `part.instance`
- `part.update()`
- `part.context.read(...)`
- `part.event.emit(...)`

---

## 8. observed state handle 的态度

Anatomy runtime access 可以成为少数允许跨 part 读取 observed state handle 的场景之一，但前提是：

- 该 state handle 已被显式 expose
- 读取方仅通过 expose 读取它

换言之：

- Anatomy 不是专门为 state 设计的读取系统
- 它只是允许在 anatomy 域内访问另一个 part 自愿暴露出来的状态观察面

---

## 9. 最小示例

```ts
const content = run.anatomy.partsOf(SELECT_FAMILY, 'content')[0] ?? null;

if (content?.hasExpose('close')) {
  const close = content.getExpose('close') as (() => void) | null;
  close?.();
}
```

---

## 10. 非目标

本文不定义：

- 跨 family 查询
- 跨 domain 广播
- 基于 anatomy 的自动调度系统
- 基于 anatomy 的自动结构补全
- 实例内部能力的无边界互访
